# GlassGradients 1.2.0

Released on 2026-04-20.

`1.2.0` is an additive feature release. It does not remove the `1.0` or `1.1` API. Existing `init`, `build`, `inspect`, root exports, component exports, adapters, recipes, effects, and legacy keys such as `animate.dfirt` remain supported.

## What Changed

- MER (Monitoring Engine Reduces) adapts `performance: auto` using runtime hints such as CPU cores, device memory, device pixel ratio, reduced motion, and data saver.
- `mainFilter` / `main-filter` separates standard glass filtering from opt-in heavy filtering.
- `mainFilter: "none"` keeps the fill layer but disables the backdrop filter.
- `main-filter: blur-filters` enables the heavier filter stack for users who explicitly want it.
- New effects: `caustic`, `liquid`, `nebula`, `iridescent`, `conic`, and `noise`.
- Engine UP controls animation timing, custom frames, movement, scale, rotation, opacity, blur, brightness, saturation, contrast, and hue rotation from `.glass`.
- Motion Blurrin adds opt-in moving circular blur fields with open spaces.
- Tailwind v4 CSS-first helpers were added without removing the Tailwind plugin API.
- A shadcn-style adapter now emits copyable component specs and CSS.
- Advanced product shells were added for data grids, charts, timelines, kanban, calendars, comboboxes, dropdowns, toasts, tabs, resizable panels, spotlight overlays, and notification centers.
- CLI now includes `lint`, `format`, `tokens`, and `inspect --score`.

## Compatibility Rules

- Manual `performance` values are respected. MER only changes performance when the config uses `performance: auto`.
- Old `mode: surface`, `mode: gradient`, and `mode: frost` behavior stays intact.
- Old commands keep working: `glassgradients init`, `glassgradients build`, and `glassgradients inspect`.
- The new shadcn adapter is a new subpath. It does not change root imports.
- Advanced components are thin shells over existing recipe fallbacks, so the defaults contract stays stable.
- Engine UP and Motion Blurrin are opt-in. Existing `animate.*` configs keep working.

## Suggested Upgrade

```bash
npm i glassgradients@latest
```

If an older lockfile still resolves a `0.x` build, update explicitly:

```bash
npm update glassgradients
```
